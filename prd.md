📝 제품 요구사항 정의서 (PRD): OOTD-Logic

1\. 프로젝트 개요

제품명: OOTD-Logic (오늘의 코디 추천 서비스)



대상: 매일 아침 "뭐 입지?"를 고민하는 대학생 및 직장인



한 줄 정의: 사용자의 옷장 데이터를 기반으로 당일 기상청 날씨 정보를 분석하여 최적의 스타일을 추천하는 AI 비서



2\. 핵심 가치 (Core Value)

개인화: 남의 옷이 아닌 '내 옷장'에 있는 옷으로만 추천.



데이터 기반: 단순 감이 아닌 실시간 온도, 습도, 강수 확률 반영.



빠른 실행: Java 17 + Spring Boot 3.5 + DynamoDB의 고성능 조합.



3\. 핵심 기능 (High Priority) ⭐⭐⭐

이 기능이 없으면 서비스가 돌아가지 않는 가장 중요한 기능입니다.



디지털 옷장 관리 (My Closet)



사용자가 자신의 옷 사진을 업로드하고 카테고리(상의, 하의, 아우터, 신발)를 지정하여 저장.



기술: AWS DynamoDB에 이미지 S3 경로 및 태그 정보 저장.



실시간 날씨 연동 (Weather API)



사용자 위치 기반으로 기상청 혹은 외부 Weather API에서 현재 기온 및 날씨 데이터를 가져옴.



날씨 기반 코디 추천 엔진 (Logic)



기온별 옷차림 가이드(예: 5도 이하면 패딩, 20도면 셔츠 등)를 알고리즘화하여 내 옷장 데이터 내에서 조합 생성.



4\. 중요 기능 (Medium Priority) ⭐⭐

서비스의 완성도를 높여주는 필수 기능입니다.



스타일 태깅 및 필터링



색상, 소재, 스타일(캐주얼, 스트릿, 오피스)별로 옷을 분류하고 검색하는 기능.



코디 기록 (OOTD 로그)



오늘 추천받아 실제 입은 코디를 기록하고 만족도를 체크하여 추후 추천 로직에 반영.



사용자 인증 및 연동



개인별 옷장 데이터를 보호하기 위한 기본 로그인/회원가입 기능.



5\. 기술 스택 (Tech Stack)

Backend: Java 17, Spring Boot 3.5.11



Database: Amazon DynamoDB (Serverless NoSQL)



Infrastructure: AWS (S3, IAM)



AI/LLM: Claude API (코디 조합 텍스트 생성 보조)
